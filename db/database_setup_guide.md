# Database Setup Guide — Fresher & Experience Agents

This guide covers the database setup needed to support the new
**FresherAgent** and **ExperienceAgent** (`ai/agents/fresher_agent.py`,
`ai/agents/experience_agent.py`), on top of the existing schema in
`db/schema.sql`.

## 1. Base setup (existing)

If you haven't already, follow `db/README.md` to spin up Postgres with
pgvector and apply `db/schema.sql`. The quick path using the repo's
`docker-compose.yml`:

```bash
docker compose up -d db
psql "$DATABASE_URL" -f db/schema.sql
```

## 2. New tables for onboarding & experience tracking

Add the following migration (e.g. `db/migrations/0001_agents_tables.sql`):

```sql
-- Track years of experience so classify_experience_level() has real data
-- instead of relying only on designation text.
ALTER TABLE team_members
    ADD COLUMN IF NOT EXISTS years_experience REAL DEFAULT 0,
    ADD COLUMN IF NOT EXISTS experience_level VARCHAR(50); -- 'fresher' | 'mid-level' | 'senior'

-- Stores plans generated by FresherAgent.create_onboarding_plan()
CREATE TABLE IF NOT EXISTS onboarding_plans (
    id SERIAL PRIMARY KEY,
    employee_id INTEGER REFERENCES team_members(id) ON DELETE CASCADE,
    project_id INTEGER REFERENCES projects(id),
    mentor_id INTEGER REFERENCES team_members(id),
    plan JSONB NOT NULL,       -- full JSON returned by the agent (day_30/60/90, etc.)
    status VARCHAR(50) DEFAULT 'active', -- active | completed | needs_review
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

-- Progress check-ins reviewed by FresherAgent.review_progress()
CREATE TABLE IF NOT EXISTS onboarding_checkins (
    id SERIAL PRIMARY KEY,
    onboarding_plan_id INTEGER REFERENCES onboarding_plans(id) ON DELETE CASCADE,
    progress_notes TEXT,
    on_track BOOLEAN,
    concerns TEXT[],
    adjustments TEXT[],
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

-- Stores assessments generated by ExperienceAgent.analyze_experience()
CREATE TABLE IF NOT EXISTS experience_assessments (
    id SERIAL PRIMARY KEY,
    employee_id INTEGER REFERENCES team_members(id) ON DELETE CASCADE,
    project_id INTEGER REFERENCES projects(id),
    seniority_tier VARCHAR(50),
    specialization_areas TEXT[],
    skill_gaps TEXT[],
    mentorship_capacity VARCHAR(20), -- low | medium | high
    stretch_assignments TEXT[],
    summary TEXT,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

-- Mentor <-> mentee pairings, produced by ExperienceAgent + FresherAgent
-- matching (find_best_mentor / suggested_mentees).
CREATE TABLE IF NOT EXISTS mentor_assignments (
    id SERIAL PRIMARY KEY,
    mentor_id INTEGER REFERENCES team_members(id) ON DELETE CASCADE,
    mentee_id INTEGER REFERENCES team_members(id) ON DELETE CASCADE,
    project_id INTEGER REFERENCES projects(id),
    reason TEXT,
    active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    UNIQUE (mentor_id, mentee_id, project_id)
);

-- Promotion-readiness assessments from ExperienceAgent.assess_promotion_readiness()
CREATE TABLE IF NOT EXISTS promotion_assessments (
    id SERIAL PRIMARY KEY,
    employee_id INTEGER REFERENCES team_members(id) ON DELETE CASCADE,
    promotion_ready BOOLEAN,
    confidence VARCHAR(20), -- low | medium | high
    supporting_factors TEXT[],
    gaps_to_address TEXT[],
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX IF NOT EXISTS idx_onboarding_plans_employee ON onboarding_plans(employee_id);
CREATE INDEX IF NOT EXISTS idx_experience_assessments_employee ON experience_assessments(employee_id);
CREATE INDEX IF NOT EXISTS idx_mentor_assignments_mentor ON mentor_assignments(mentor_id);
CREATE INDEX IF NOT EXISTS idx_mentor_assignments_mentee ON mentor_assignments(mentee_id);
```

Apply it with:

```bash
psql "$DATABASE_URL" -f db/migrations/0001_agents_tables.sql
```

## 3. Populating `experience_level`

Backfill existing rows using the same heuristic the agents use in
`ai/agents/tools.classify_experience_level`:

```sql
UPDATE team_members
SET experience_level = CASE
    WHEN designation ILIKE ANY (ARRAY['%fresher%','%intern%','%graduate%','%trainee%','%junior%'])
        THEN 'fresher'
    WHEN designation ILIKE ANY (ARRAY['%senior%','%staff%','%principal%','%lead%','%architect%'])
        THEN 'senior'
    WHEN years_experience >= 5 THEN 'senior'
    WHEN years_experience < 1.5 THEN 'fresher'
    ELSE 'mid-level'
END;
```

## 4. Routing logic in the backend

When wiring these agents into `backend/app/api/team_members.py` (or a new
`backend/app/api/onboarding.py`), route each team member by
`experience_level`:

```python
from ai.agents.fresher_agent import FresherAgent
from ai.agents.experience_agent import ExperienceAgent
from ai.agents.tools import classify_experience_level

level = classify_experience_level(member.designation, member.years_experience)

if level == "fresher":
    result = FresherAgent().create_onboarding_plan(...)
else:
    result = ExperienceAgent().analyze_experience(...)
```

Persist the returned JSON into `onboarding_plans` or
`experience_assessments` respectively, and use `mentor_assignments` to
record whichever mentor/mentee pairing the agents suggest.

## 5. Environment variables

No new environment variables are required — both agents reuse
`GOOGLE_API_KEY` from `backend/.env` (same variable used by
`backend/app/core/ai_agents.py`). If it's unset, both agents fall back to
deterministic mock responses so local development keeps working end-to-end.
